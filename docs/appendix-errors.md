# 附录 · 常见报错速查

> 看到报错别怕。把红字截图发给混元3，加一句「我是新手，帮我修并解释」，AI 就能搞定。下面列最常见的几种，方便你对号入座。

---

## 1. `url not in domain list` / 不在合法域名列表

- **含义**：小程序联网请求必须走备案过的 HTTPS 域名，测试号没配。
- **临时办法**：让 AI 把请求改成返回本地模拟数据（见 Prompt D）。
- **正式解决**：公众平台「开发设置 → 服务器域名」加你的 HTTPS 域名。

---

## 2. `app.json` 报错 / 页面找不到

- **含义**：`app.json` 里 `pages` 配置的路径和实际文件对不上。
- **让 AI 说**：「打开报 app.json 错误，请检查 pages 数组里的路径是否都有对应文件。」
- **预防**：每次加页面，确认 AI 同时把路径写进了 `app.json`。

---

## 3. 白屏 / 页面空白

- **常见原因**：首页路径不对、某个 `.js` 语法错、数据没初始化。
- **排查**：看 Console 红字第一行，定位文件。让 AI 整体体检（见 Prompt G）。

---

## 4. 真机调试提示「该功能测试号不可用」

- **含义**：微信登录、支付等能力测试号不支持。
- **办法**：让 AI 做「未授权/不可用」的友好占位（见第 5 章 5.5）。换成正式 AppID 后可用。

---

## 5. 样式错乱 / 不同手机显示不一致

- **原因**：用了 `px` 而非 `rpx`（小程序自适应单位）。
- **让 AI 说**：「部分元素尺寸不对，帮我统一改成 rpx 适配不同屏幕。」

---

## 6. 上传代码提示「上传失败 / 没有权限」

- **含义**：当前 AppID 不是你账号下的，或用了测试号（测试号不能上传）。
- **解决**：换成你自己的正式 AppID 重新导入项目。

---

## 7. Git 推不上去：`Host key verification failed`

- **含义**：本机第一次连 GitHub，没信任它的主机密钥。
- **解决**：按提示输入 `yes` 接受；或手动 `ssh-keyscan github.com >> ~/.ssh/known_hosts`。

---

## 8. Git 推不上去：权限拒绝

- **含义**：没配置 SSH 公钥，或没登录。
- **解决**：把本机 SSH 公钥加到 GitHub 账户的 SSH Keys；或改用 HTTPS 方式（会要账号密码/令牌）。

---
