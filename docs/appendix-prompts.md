# 附录 · 常用 Prompt 速查

> 直接复制粘贴给混元3 用。把 `<尖括号>` 里的内容换成你自己的。

---

## A. 初始化项目

```
我要做一个微信小程序，主题是「<微信小程序教程>」，用原生微信小程序（不要框架）。
请帮我生成最基础的项目结构，包含一个首页：
- 顶部搜索框
- 分类卡片（按钮类/表单类/进度类/反馈类）
- 底部最近浏览列表
风格简洁，微信绿为主色。请把生成的文件列出来并说明每个文件作用。
```

---

## B. 加一个页面

```
在现有小程序里新增一个「<商品列表>」页面：
- 卡片式商品，带图片、标题、价格
- 下拉刷新、上拉加载更多
- 点击进入详情（先占位）
请用原生小程序实现，告诉我改了哪些文件、加在 app.json 的哪里。
```

---

## C. 调试：描述报错

```
我的小程序报这个错（贴 Console 红字截图或文字）：
<粘贴报错>
请告诉我可能原因，并给出修改后的代码。我是新手，请顺便用大白话解释改了啥。
```

---

## D. 联网功能绕过域名限制（测试号）

```
我的 wx.request 报 "url not in domain list"，我现在用测试号。
请先把网络请求改成返回本地模拟数据，让页面正常显示；并标注清楚哪里将来要换成真实接口。
```

---

## E. 硬件能力

```
我想加一个「<扫码>」功能，调用微信扫码 API。
请实现：点按钮调起扫码，把结果展示出来。
注意：测试号/模拟器可能不支持，请做「不支持时的友好提示」。
另外如果使用传感器，请在页面离开时停止监听。
```

---

## F. 改成自己的产品

```
把这个小程序从「控件教程」改造成「<我的菜谱>」：
- 首页改成菜谱分类
- 每个菜有图片、食材、步骤
- 去掉题库和硬件模块
只改内容和页面，保留现有工程结构。
```

---

## G. 整体体检

```
请帮我检查整个小程序：有没有白屏风险、样式是否用 rpx 适配、有没有没释放的定时器/监听、有没有调用需声明隐私的接口。列个清单给我。
```
